Inputs Inputs for grid rows Input data for grid. Accepts an array of Objects. Make sure that each object in the rows have same properties as that of assigned columns in cols array of grid
columns Input columns for grid. Accepts an array of Objects. Each object in this array should contain a key-value pair with key as "name", value being the property to be displayed.

Each column can accept a custom cell Template (cellTemplate), text align (textAlign) , editable properties,. If these properties are set, the column will be rendered according to cell template and text-align values. Else, normal cell template with center aligned columns will be rendered. If editable is set to true, then double clicking on the column enables the editing field
selectionType

Single-select/multi-select option for grid. Accepts a string. If selectionType is set to 'multiple', checkboxes will be rendered in the first column. Else if the selectionType is set to 'single', radio buttons will be rendered. The selection made will be emitted through the 'selectedData' method. Removing the selectionType property results in just a scrollable list. Find the difference between this with the Basic and Multi-select examples from the left panel.
gridHeight

The height of the grid in pixels. Accepts a number which will then be used to set the height of the grid in pixels.
gridWidth

The width of the grid in pixels. Accepts a number which will then be used to set the width of the grid in pixels.
headerRowHeight

The height of header-row. Accepts a number. This property sets the height of header-row of the grid in pixels.
rowHeight

The height of each row. Accepts a number. This property sets the height of each row in the grid in pixels.
forceColumnWidth

Accepts a number. If set to true, you need to specify custom width for each of the columns. If set to false, width for each column will be same. Visit the Custom Width example from the left bar
loadType

Accepts a string. If set to 'pagination', renders the grid with pagination at the bottom. By default, 'virtualScroll' is assigned internally
showToolbar

Accepts a boolean. If set to true, renders a toolbar which contains search functionality, csv, excel download of whole grid data.
toolbarOptions

As of now, takes three properties

  • searchPlaceholder: Accepts a string. It is the placeholder text to be shown in the search input textbox
  • csvDelimiter: Accepts a string. In most of the cases, csv files come with '|' or ',' as delimiters. If not mentioned, ',' is taken as default
  • fileName: Accepts a string. This is the name that will be appended while downloading the file. If not mentioned, file will be named as 'gridData.csv' by default. And dont forget to add an 'csv.png' image to ur assets folder.
preSelectedRows

Accepts an array of objects. Just make sure that the array passed to this property is a subset of the rows which is assigned to datagrid
styles Accepts an object of the following properties.

headerBgColor:string,

headerFontColor:string,

showInsideBorders:boolean,

toolbarBgColor:string,

paginatorBgColor:string,

paginatorFontColor:string,

showEvenOdd:boolean

toolsPanelHeight:number

showDownloads:boolean
Output Events emitted by grid.

If the selectionType is set to 'multiple' or 'single', then the selected rows will be emitted from the event 'selectedData'. Have a look at the Single Select and Multi Select examples
Extras List of Features provided by ngx-data-grid

Sorting - The grid comes with merge-sort algorithm. You can sort any column either ascending or descending just by clicking on the column name.

Virtual Scroll - The grid comes with virtual-scroll by default. This technique helps to load large datasets into the grid without any perfomance lag

Pagination - The grid comes with pagination feature. To use pagination instead of virtual scroll, loadType property has to be set to 'pagination'

Cell Templates - The grid comes with the ability to accept cell-templates for any/all of the columns. Have a look at the examples of Editable and Template Ref

Integrated Toolbar - The grid has an integrated toolbar which contains search functionality, Csv, Excel downloads on whole grid data. Have a look at the example of Tool Bar
